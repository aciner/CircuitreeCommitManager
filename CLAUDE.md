# CLAUDE.md

This file provides guidance when working with the FRC 1599B robot codebase (Rebuilt-new).

## Project Overview

This is a WPILib-based Java FRC robot project for team 1599B, built on the command-based framework.
The robot features a swerve drivetrain, shooter turret, intake, elevator/climber, and Limelight vision.
Autonomous paths are managed via PathPlanner and Choreo.

## Build & Run Commands

```bash
# Build the project
./gradlew build

# Deploy to robot (robot must be connected)
./gradlew deploy

# Simulate the robot in the WPILib simulator
./gradlew simulateJava

# Run tests
./gradlew test
```

> Always use `./gradlew`, never a bare `gradle` command.

## Architecture

### Entry Points
- **`Robot.java`** — Extends `TimedRobot`. Bootstraps `RobotContainer`, runs the `CommandScheduler` every loop, and handles mode transitions (auto, teleop, test). Also updates the `Field2d` pose display each teleop loop.
- **`RobotContainer.java`** — Wires together all subsystems, binds controller buttons to commands, registers PathPlanner named commands, and returns the selected auto.

### Subsystems (`src/main/java/frc/robot/subsystems/`)
| Class | Hardware | Purpose                                                   |
|---|---|-----------------------------------------------------------|
| `CommandSwerveDrivetrain` | 4x TalonFX drive + steer, Pigeon 2 | Swerve drive, odometry, PathPlanner integration — **do not edit** |
| `Turret` | TalonFX ×3 (left/right shooter, rotator), CANcoder, LinearServo (hood) | Flywheel shooter, turret rotation, hood angle             |
| `Elevator` | TalonFX (motor ID 31) | Climber elevator with MotionMagic position control        |
| `intake` | TalonFX (deploy + feed motors), CANcoder | Deploy/retract intake arm, run feed wheels                |
| `Trigger` | TalonFX (index motor) | Kicks game pieces into the shooter                        |
| `LimelightHelpers` | — | Auto-generated Limelight helper library — **do not edit** |

### Commands (`src/main/java/frc/robot/commands/`)
Commands follow WPILib's command-based pattern. Each command declares its required subsystems in the constructor via `addRequirements()`.

| Command | What it does |
|---|---|
| `AutoShoot` | Waits for turret to reach speed, then feeds with a delay timer |
| `AutoTurret` | Uses odometry zones + Limelight to aim and shoot autonomously |
| `SpinToDistanceSpeed` | Looks up shooter speed from the `InterpolatingDoubleTreeMap` based on Limelight distance |
| `TurnTurret` | Uses Limelight TX + PID to auto-rotate the turret toward a target |
| `ManualTurret` | Rotates turret from a joystick axis input |
| `Shoot` | Spins shooter then feeds — used in teleop trigger binding |
| `ReverseShoot` | Reverses both shooter and trigger (unjam) |
| `ZeroTurret` | Slowly drives rotator until current spike, then zeros encoder |
| `LineUpClimb` | Aligns drivetrain to cage using Limelight before climb |
| `ClimbUp` / `climbDown` | Drives elevator to top/bottom setpoint |
| `DeployIntake` / `RetractIntake` | Move intake arm to deploy/home position |
| `RunIntake` / `RunOuttake` | Run feed wheels in/out |
| `ManualDeploy` | Manual power control for deploy motor |
| `ToggleHood` | Flips hood servo between up and down positions |
| `StopTurretWheels` | Stops shooter flywheel motors |

### Utilities (`src/main/java/frc/robot/util/`)
- **`Limelight.java`** — Reads from the `limelight-turret` NetworkTable. Key methods:
    - `getTX()` / `getTY()` / `getTA()` — raw Limelight values
    - `getDistance()` — converts target area (`ta`) to distance in inches using a calibrated power curve: `46.39986 * ta^-0.4918478`. Returns 0 if no valid target.
    - `getTurretSpeed()` — PID output to center the turret on target (setpoint = 0 tx)
- **`LinearServo.java`** / **`HiTecServo.java`** — Servo wrappers used for the hood mechanism
- **`ButtonMap.java`** — Unused placeholder for future arcade button panel support

### Generated (`src/main/java/frc/robot/generated/`) — **do not edit**
- **`TunerConstants.java`** — Auto-generated by Phoenix Tuner X. Contains swerve module offsets, drive gains, and `createDrivetrain()`. **Do not hand-edit.**
- **`Pigeon.java`** — Auto-generated IMU config.

### Configuration
- **`Constants.java`** — All tunable values. Key inner classes:
    - `CAN_IDS` — All CAN bus device IDs (CAN bus name: `"FRC 1599B"`)
    - `Turret` — Angle limits, shooter speed thresholds, distance-to-speed lookup values (`distClose/Mid/Far`, `speedClose/Mid/Far`), `autoShootFeedDelay`
    - `Intake` — Deploy/home positions, speed
    - `Drive` — Max speed/angular rate, speed divisor toggle
    - `Climber` — Current threshold for hard-stop detection
- **`Telemetry.java`** — Logs drivetrain state to SmartDashboard/AdvantageScope each loop.

## Key Patterns & Conventions

### Shooter Speed Lookup
`Turret` uses WPILib's `InterpolatingDoubleTreeMap` to map Limelight distance → flywheel speed. Add or adjust data points in `Constants.Turret` (`distClose`, `distMid`, `distFar`, etc.), not inside `Turret.java` directly.

### Safety Checks
`Turret.isSafe(output)` must return `true` before any rotator movement is applied. It enforces `Constants.Turret.minAngle` and `maxAngle`. Never bypass this check.

### Motor Configurations
Motors are configured with Phoenix 6's `TalonFXConfiguration` / `Slot0Configs` in each subsystem constructor. When changing PID gains, prefer updating `Constants.java` values and re-applying configs rather than hardcoding numbers inline.

### Neutral Modes
- Shooter flywheel motors: **Coast** (so they spin down gradually)
- Rotator / elevator: **Brake** (hold position when idle)
- Deploy motor: switches between **Coast** (during movement) and **Brake** (when retracted)

### Autonomous
- Auto routines are built in PathPlanner and selected via `SendableChooser` on SmartDashboard.
- Choreo trajectory files live in `src/main/deploy/choreo/`.
- Named commands are registered in `RobotContainer.mapEventsToCommands()`. Any new auto action must be registered there before it can be used in PathPlanner.

## Known Issues & TODOs
- Several command class names use `camelCase` (`runIntake`, `deployIntake`, `climbDown`) instead of the Java convention of `PascalCase`. Rename when touching those files.
- `Constants.Speed`, `Constants.TriggerPositions`, and `Constants.HoodPositions` are empty placeholder classes.
- Significant dead/commented-out code in `RobotContainer.configureBindings()` — clean up when button layout is finalized.
- `Robot.java` contains an unused inner class `Region2d` and duplicate `HashMap` import.
- `AutoShoot` has a hardcoded speed threshold of `30` RPS with a `//TODO` comment — this should be moved to `Constants.Turret`.
